What is it?

I created a mock website (like the name suggests) of wikihow.com. I thought it would be fun and funny if users would "accidently" stumble upon my website looking for wikihow advices but only end up with really bad advises. My web app will let the user ask any question that they wish to be answered like wikihow.com, but instead generate a page of somewhat random advices (explained below). In order to make the wikihow article feel and seem like a real one at first glance, I scraped real wikihow articles and saved them into a table, then have my webapp pick senteces and pictures from this table.

Generating articles

Because I wanted to "trick" the user to thinking my website is wikihow.com, I thought it would be interesting if the articles would gradually get more random as the user reads on. Therefore, I used NLTK, a natural processing library to look for similar words from the user's query in the database when generating the articles. However, this did not turn out as well as I expected, since I did not have a efficient way to search for similar words/sentences through the database. Therefore, for now I linked some pre-generated pages from the home page of my website that attempted to create gradually more random articles.
